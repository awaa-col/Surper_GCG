# 概念级攻击：深度交叉研究报告

> 核心命题：**概念是不变的，变的只是不同模型的表达。因此攻击应当针对概念本身，而非绑定特定层。**

---

## 一、关键论文发现

### 📄 1. CAVGAN — ACL 2025（越狱+防御统一框架）

**核心方法**：用 GAN 学习 LLM **内部安全决策边界**。
```
Generator G: 生成"安全概念激活向量"(SCAV)方向的扰动
Discriminator D: 判断扰动后的嵌入是否仍在"安全侧"
→ GAN 收敛后，G 学会了恰好穿越安全边界的最小扰动
```

**关键数据**：
- 跨 3 个主流 LLM 平均越狱成功率：**88.85%**
- 防御成功率：**84.17%**（同一个框架，反过来用就是防御）

**与我们实验的交叉验证**：
| CAVGAN 发现 | 我们的实验验证 |
|------------|-------------|
| 安全概念在内部空间**线性可分** | ✅ Exp01: PCA top1 在 L8 解释 72% 方差 |
| 越狱 = 将表征**推过决策边界** | ✅ Exp01b: L17消融 = 将激活在拒绝方向的投影归零 |
| GAN 不需要指定特定层 | ⚠️ 我们目前绑定了 L17/L18 |

> **🔥 关键差异**：CAVGAN 的 GAN 自动学习"在哪一层、什么方向、推多远"——这正是我们缺少的"概念自适应"机制！

---

### 📄 2. SCAV — Xu et al. 2024（安全概念激活向量）

**核心方法**：量化 LLM 在每一层判定输入"有害/无害"的概率。
```
对每层 l，训练线性分类器 f_l：
  P(malicious | h_l) = σ(w_l · h_l + b_l)

SCAV = w_l（分类器权重 = 该层的安全概念方向）
```

**关键发现**：
- 安全/有害的线性可分性**从约 L10 开始出现**
- SCAV 能**自动选择最该攻击的层 + 计算最优扰动幅度**

**与我们实验的交叉验证**：
| SCAV 发现 | 我们的实验验证 |
|----------|-------------|
| 线性可分从 ~L10 开始 | ✅ Exp01: stability从L7开始>0.95，PCA在L8-L9最集中 |
| 自动选层 | ⚠️ 我们手动选了L17（通过逐层消融试出来的） |
| 最优扰动幅度可计算 | ⚠️ 我们的α是手动试的（5, 10, 20） |

> **🔥 SCAV 的核心优势**：它把「选哪层」和「推多远」变成了**可求解的优化问题**，而不是我们现在的手动试错。

---

### 📄 3. 表征空间分析 — "Towards Understanding Jailbreak Attacks"

**核心发现**：
> 成功的越狱攻击，其本质是将有害 prompt 的内部表征**平移（shift）到无害 prompt 的表征区域**。

```
正常状态：h(harmful) 在"拒绝侧"
攻击后：  h(harmful + adv_suffix) 被推到"合规侧"
         ↑ 对抗 suffix 的 Attention 产生的影响
```

**与我们实验的交叉验证**：
| 论文发现 | 我们的实验验证 |
|---------|-------------|
| 对抗suffix在表征空间造成方向性平移 | ✅ 我们提取的 r_exec 就是这个平移方向 |
| 平移方向与安全概念方向对齐 | ✅ Exp06b: cos(c_dir, harm_concept) ≈ 0.99 |
| 平移幅度决定了是否穿过决策边界 | ✅ Exp07: persona steer α=5最优，α过大反而退化 |

## 二、核心洞察：Token 如何影响概念？

综合以上论文，Token→概念的影响链条如下：

```
输入 Token
  ↓ Embedding Layer
Token 向量 (x_1, x_2, ..., x_n)
  ↓ Self-Attention (所有层)
每个 Token 通过 Attention 影响其他 Token 的表征
  ↓ 残差流累积
在某些关键层(~L10+)，"安全概念"被激活
  ↓ 线性决策面
模型根据激活在安全概念方向的投影做出"拒绝/合规"决定
```

**对抗 Token 的作用机制**：
1. 对抗 Token 被 Embedding 成向量
2. 通过 Attention 机制，这些向量**向所有其他 Token 广播干扰信号**
3. 在关键层，这种干扰**等效于沿安全概念方向做了一次反向推力**
4. 推力足够大时，整体表征穿过线性决策面 → 越狱

> **这就是为什么对抗 suffix 看起来是乱码，但确实有效——它不需要在语义上有意义，只需要在表征空间中产生正确方向的推力！**

## 三、概念级攻击方案设计（与我们实验的整合）

### 原方案（层绑定型）：❌ 泛用性差
```python
Loss = cos(h_L17, r_exec) + cos(h_L18, persona_dir)  # 绑死了层号
```

### 新方案（概念自适应型）：✅

```python
# 步骤 1：概念方向自动提取（SCAV 方法）
for layer in ALL_LAYERS:
    scav[layer] = train_linear_probe(
        harmful_activations[layer],
        safe_activations[layer]
    )

# 步骤 2：概念级 Loss（不绑定层，梯度自动权衡）
def concept_loss(model, input_tokens):
    L = 0
    for layer in ALL_LAYERS:
        h = get_hidden_state(layer)
        # 安全概念投影（SCAV 权重）
        safety_score = scav[layer].weight @ h
        # 目标：让每一层都觉得这个输入"无害"
        L += safety_score  # 最大化"安全得分" = 骗过所有层
    return L

# 步骤 3：GCG 迭代（梯度驱动的 Token 搜索）
for step in range(N_STEPS):
    loss = concept_loss(model, harmful_prompt + adv_tokens)
    grad = compute_gradient(loss, adv_tokens)  # 对 Token 求梯度
    adv_tokens = greedy_coordinate_update(adv_tokens, grad)
```

### 与旧方案的关键区别

| 维度 | 旧方案(层绑定) | 新方案(概念级) |
|------|-------------|-------------|
| 层选择 | 手动指定 L17, L18 | **全层扫描，梯度自动聚焦** |
| 方向选择 | mean-diff 提取的单一向量 | **每层训练线性探针（SCAV）** |
| 扰动幅度 | 手动试 α=5,10,20 | **梯度自动优化** |
| 跨模型迁移 | ❌ 层号不通用 | ✅ 概念是通用的，只需重新训练探针 |
| 是否需要 hook | ✅ 需要修改中间层 | ❌ **纯 Token 输入攻击** |

## 四、为什么这对黑盒攻击有希望？

1. **概念的跨模型不变性**（柏拉图假说）：
   - "安全/危险"这对概念在所有大模型内部都存在
   - 只是坐标轴不同，但局部拓扑关系一致
   - 在小模型上找到能翻转"安全概念"的 Token → 大概率在大模型也能引起类似的概念翻转

2. **对抗 suffix 的迁移性已被证实**：
   - DeGCG 框架证明了跨模型迁移的可行性
   - CAVGAN 在 3 个不同 LLM 上成功率 88.85%

3. **我们的实验提供了额外优势**：
   - 我们知道安全概念从 ~L8 开始凝聚（比 SCAV 的 ~L10 更早）
   - 我们知道身份概念在最后 1/3 层（L18+）
   - 这些知识可以指导 Loss 函数的层权重分配
